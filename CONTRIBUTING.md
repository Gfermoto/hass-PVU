# Contributing

Спасибо за интерес к проекту `hass-PVU`.

## Как внести изменения

1. Сделайте форк и создайте ветку:
   - `feat/<short-name>` для новых функций;
   - `fix/<short-name>` для исправлений;
   - `docs/<short-name>` для документации.
2. Внесите изменения небольшими логическими шагами.
3. Обновите документацию, если меняется поведение (`README.md`, `CHANGELOG.md`).
4. Перед PR проверьте:
   - blueprint импортируется в Home Assistant;
   - нет явных регрессий по ключевым сценариям (см. `docs/RELEASE_CHECKLIST.md`).
5. Откройте Pull Request и заполните описание:
   - что изменено;
   - почему это нужно;
   - как проверяли.

## Стиль коммитов

Рекомендуемый формат:

- `feat: ...` — новые возможности;
- `fix: ...` — исправления;
- `ui: ...` — изменения интерфейса/UX;
- `docs: ...` — документация;
- `chore: ...` — сервисные изменения.

Сообщение коммита должно кратко отражать **причину** изменений.

## Что важно для blueprint

- Не ломать совместимость с `pvu_min.yaml` (стабильная база).
- Новые эксперименты и расширения вносить в `pvu.yaml` (develop-ветка).
- Любые потенциально рискованные изменения (антизаморозка, режимы HVAC/FAN, условия away) сопровождать явным описанием в PR.

## Как сообщать о багах

Используйте шаблон issue:

- приложите фрагмент automation trace;
- укажите версии Home Assistant;
- приложите ключевые входные параметры blueprint.
